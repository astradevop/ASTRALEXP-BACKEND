# ASTRALEXP AI/ML Growth Roadmap

## Goal

Convert ASTRALEXP from a basic prompt-to-Google-API expense logger into a stronger Python full-stack + AI/data project.

Interview positioning:

> ASTRALEXP is a mobile-first expense manager where I use Django REST Framework for the backend, Expo React Native for the frontend, and AI/data tools selectively for natural-language expense extraction, validation, analytics, and spending insights.

Important note:

> AI should support difficult parsing, testing, analytics, and edge cases. It should not look like the whole project was generated by AI.

## Feature 2: AI Expense Parser With Backend Validation

Improve the current prompt flow.

User input example:

```text
Paid 350 for petrol using GPay yesterday evening
```

Expected structured output:

```json
{
  "amount": 350,
  "category": "Fuel",
  "payment_method": "GPay",
  "expense_time": "2026-04-20T18:00:00",
  "note": "petrol"
}
```

Backend must validate:

- Amount is numeric
- Category is allowed or normalized
- Payment method exists or asks user to create it
- Date/time is valid
- Required fields are not empty

Interview value:

> I did not directly trust the AI response. I used AI for extraction, then validated the JSON in Django before saving it to the database.

## Feature 3: Rule-Based Parser + AI Fallback

Create a safer parsing pipeline.

Recommended flow:

1. Try simple Python rule-based parsing.
2. If confidence is low, call Gemini/OpenAI-style API.
3. Validate the AI response.
4. If fields are missing, ask the user a follow-up question.
5. Save only after required data is complete.

Example:

```text
User: Spent 200
System: Which payment method did you use?
```

Interview value:

> I designed the AI feature with fallback and validation instead of depending blindly on the model.

## Feature 4: Spending Prediction

Possible features:

## Feature 5: Unusual Expense Detection

Detect abnormal or unusually high expenses.

Simple logic:

- Calculate average spending per category

- Mark expenses as unusual if they are much higher than normal

Libraries:

- pandas
- numpy
- scikit-learn optional

Interview value:

> I implemented basic anomaly detection to flag unusual spending behavior using statistical analysis.
